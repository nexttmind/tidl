 External transaction integration
You charged the patient on YOUR gateway (Braintree, Adyen, Worldpay — whatever you
already integrated). PRX never calls back to that gateway; we just record the transaction for
audit, receipts, refund reconciliation, and the encounter's payment status. setTransaction is a
one-call shortcut for this — equivalent to setPayment({ mode: 'reference_captured', ...}) .
The integration pattern
// 1. Charge the patient on YOUR gateway (typical Stripe / Braintree / etc.)
const result = await stripe.paymentIntents.create({
  amount: 29900,         // in cents
  currency: 'usd',
  payment_method: paymentMethodId,
  confirm: true
});
// 2. Record the transaction in PRX so it shows up on the order + statement
PrescribeRx.setTransaction({
  transaction_id: result.id,            // 'pi_3PqK...'
  amount:         result.amount / 100,   // $299.00
  gateway:        'stripe',           // free-form (record-only)
  captured_at:    new Date().toISOString()
});
// That's it. The wizard's payment step shows "Paid" and the encounter's
// resulting Order has a captured Transaction row. PRX never calls Stripe.
// Equivalent — long form via setPayment
PrescribeRx.setPayment({
  mode:    'reference_captured',
  gateway: 'stripe',
  transaction: {
    transaction_id: result.id,
    amount:         result.amount / 100,
    captured_at:    new Date().toISOString()
  }
});

Verifying the recorded transaction
After the encounter is created, fetch the resulting order and check its transactions. The
external_transaction_id column holds the ID you passed in.
curl "…/api/v1/orders/{ORDER_ID}/transactions" \
  -H "Authorization: Bearer $TOKEN"
