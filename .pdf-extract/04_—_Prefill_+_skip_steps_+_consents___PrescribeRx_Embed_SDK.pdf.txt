 Full prefill + skip steps + pre-accepted consents
White-label flow: your platform already has the patient on file (registered, ID-verified,
consented). Skip steps the wizard would otherwise re-collect, prefill the patient block, and
pass through the structured ID metadata + consent acknowledgements you captured
upstream.
The integration pattern
// 1. Pre-fill demographics + answers (keyed by field slug)
PrescribeRx.prefill({
  first_name:    'Jane',
  last_name:     'Smith',
  email:         'jane.smith@example.com',
  date_of_birth: '1988-03-22',
  phone:         '555-555-9876',
  current_weight_lbs: 185,
  weight_goal:   'lose_15_lbs'
});
// 2. Skip steps you've already handled upstream
PrescribeRx.setSkipSteps([
  'identification',    // already verified during signup
  'payment-step',      // charged on our gateway already
  'review-step'        // fast-path through review
]);
// 3. Pass through ID metadata captured during your signup flow
PrescribeRx.setIdentification({
  id_type:   'drivers_license',
  id_number: 'D1234567',
  id_state:  'GA'
});
// 4. Pre-mark consents the patient already accepted on your platform
PrescribeRx.setConsents([
  {
    consent_type:    6,                    // Telehealth (see /api/docs/field-
    consent_text:    'I consent to telehealth consultation.',
    consent_version: '2.1',
    consented_at:    '2026-04-15T10:32:00Z',
    signature_method: 'click'
  },
