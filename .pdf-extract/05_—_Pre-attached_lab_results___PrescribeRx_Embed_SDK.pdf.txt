 Pre-attached lab results
Encounter type with requires_labs=true ? Don't make the patient wait for a fresh draw if they
already have recent results. Pass them via setLabResults and the wizard skips its auto-order-
labs branch + persists the PDFs as patient_documents linked to the chart and encounter.
The integration pattern
// 1. Attach prior lab results — base64 OR HTTPS URL fetched server-side
PrescribeRx.setLabResults([
  {
    name:         'TSH 2026-04',
    lab_type:     'TSH',
    collected_at: '2026-04-15',
    pdf_base64:   'JVBERi0xLjQK...'     // preferred
  },
  {
    name:         'Comprehensive Metabolic Panel',
    lab_type:     'CMP',
    collected_at: '2026-04-15',
    pdf_url:      'https://your-cdn.example.com/labs/abc123.pdf'
  }
]);
// 2. (Optional) attach other supporting documents alongside
PrescribeRx.setDocuments([
  { type: 'lab_result_pdf', filename: 'thyroid_panel.pdf', file_base64: '...' 
]);
// On submit, the wizard checks each required panel against the supplied
// lab_type markers. All required panels covered → labs_satisfied=true and
// the auto-lab-order branch is skipped. Missing panels → ONLY those are
// auto-ordered. See /api/docs/field-mapping#documents for the full schema.

still needed. The endpoint returns missing_biomarker_ids[] per panel — pass through to your UI so the
patient sees "we still need a free T4" instead of a generic "labs required" prompt.
